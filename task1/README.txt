Задание:
    A) Шар. Разработать классы Точка и Овал. Создать методы и тесты: вычисления
    площади поверхности шара, объема шара, соотношения объемов получаемых в
    результате рассечения шара координатной плоскостью; является ли объект
    шаром; касается ли шар любой из координатных плоскостей.
    B) Площади, объемы, Периметры фигур должны храниться в объекте класса-
    регистратора. Любое изменение параметра фигуры должно вызывать перечсчет
    соотв. значений в классе регистраторе.
Были разработаны классы для разных задач:
    1) Entity-классы - объекты сущностей: Сфера, Точка, Плоскость
    2) Action-классы - классы для работы с объектами (подсчет площади и т. п);
    Разработан один класс: SphereAction.
    3) Readers: для чтения из файла данных
    4) Recorders: хранят заведомо посчитаные значения фигур (явл. наблюдателями)
    При изменении наблюдаемого класса, их значения тоже меняются.
    5) Exceptions: исключения
    6) Repository: для хранения данных.
    Данные хранятся в виде пары. Пара включает в себя собственно
    сам класс Сферы и его Recorder, который хранит значения объема
    и поверхностной площади сферы. До этого я не реализовывал такое хранение, т. к.
    в задании этого не было сказано. Не уверен насчет правильности такого решения.
